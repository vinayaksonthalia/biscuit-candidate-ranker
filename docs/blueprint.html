Solution Overview 02/11

Proposed Solution

Biscuit is a highly optimized, deterministic, CPU-only candidate ranking engine. It streams candidate records line-by-line using Python generators, eliminating high RAM peaks and scaling gracefully.

  • Staged Pipeline Architecture: Discards honeypots instantly, scores 6 core dimensions, applies suspicion multipliers, and sorts deterministically.
  • Zero Network Latency: Resolves logic entirely locally without any live LLM API calls, ensuring 100% speed, safety, and reproducibility.
🚀 What Differentiates Our Approach?

Traditional systems rely on raw keyword counting, which is easily gamed by candidates padding skills.

  • Trajectory & Culture Alignment: We evaluate career progression, penalizing services-only tenure while favoring product ML builders.
  • Data Integrity Cross-Check: We programmatically calculate a candidate's "Robust YoE" and penalize chronological discrepancies.

Ranking Methodology 04/11

Multi-Dimension Scoring

Candidate scores are normalized from 0.0 to 1.0 using six weighted dimensions:

  • Career Relevance (30%): Product tenure ratio, ML keywords in titles.
  • Skills Match (25%): Mandatory Must-haves vs. Fringe domain penalties.
  • Experience Band (15%): Smooth piecewise curve, flat peak across the 6–8 YoE band.
  • Behavioral Signals (15%): Recruiter response and profile activity.
  • Location & Logistics (10%): Residency (Noida/Pune) and notice period.
  • Education Quality (5%): Tier-1/Tier-2 degrees & major relevance.
🧮 Heuristics & Multi-Signal Blend
  • The Skills Gate: Imposes a mandatory threshold on technical scores; non-technical candidates cannot rank highly due to location/logistics alone.
  • Suspicion Penalty Multipliers: Instead of immediate removal, borderline inconsistencies (e.g. minor timeline overlaps) discount the score by 5-30%.
  • Deterministic Tie-Breaking: Sorts descending by score, ascending by candidate ID. Ensures 100% reproducible rankings.

Explainability & Validation 05/11

How Decisions are Explained

  • Fact-Grounded Reasoning: Every description is built from actual profile fields (role, company, YoE, specific skills, degree relevance).
  • Zero Hallucinations: The engine uses zero generative text APIs, making justifications 100% factual and deterministic.
  • Deterministic Variety: Computes MD5 hash of candidate ID to seed sentence templates. The same candidate gets identical reasoning, but text varies across candidates.
🛡️ Defending Against Suspicious Profiles
  • Definite Honeypot Filters: Immediate removal of candidates with impossible data (zero-duration expert skills, single job duration > total self-reported YoE, or job dates exceeding calendar limit by >12 months). 42 caught.
  • Robust YoE Verification: Calculates experience from job spans rather than trusting raw text values, bypassing self-reported padding.

Pipeline Workflow 06/11

Complete Job-to-Candidate Workflow

1. Ingest & Stream

Stream `candidates.jsonl` record by record. Parses details dynamically, scoring the full 100K pool within a ~2 GB peak footprint — comfortably under the 16 GB limit.

2. Filter & Clean

Validate candidate timestamps and durations. Discard impossible profiles (honeypots) and compute suspicion discounts for borderline cases.

3. Dimension Score

Apply the scoring matrix across 6 dimensions, pass candidates through the technical skills gate, and compute the final weighted index.

4. Sort & Rank: Sort candidates descending by score and break ties using ID sorting.
5. Write & Validate: Generate fact-grounded reasonings, output to `submission.csv`, and run strict format checks.

Architecture 07/11

Modular Architecture

  • loader.py: Memory-efficient streaming JSONL loader.
  • filters.py: Hard honeypot filtering and suspicion check.
  • scorer.py: 6-dimensional weighted scoring system.
  • ranker.py: Deterministic sorting and justification builder.

Design Philosophy: We deliberately chose simple, transparent, deterministic Python rules over heavy neural models to guarantee sub-10 second, CPU-only execution with zero network requirements.
